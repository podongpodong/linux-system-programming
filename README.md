## 과제 #1 - ssu_cleanup
### 개요
사용자가 지정한 디렉터리를 트리 구조로 시각화하고, 파일을 확장자 기준으로 분류할 수 있도록 구현한 Linux 기반 CLI 파일 관리 프로그램입니다.  
경로 정규화, 디렉터리 순회, 파일 메타데이터 조회, 예외 처리, 중복 파일명 처리 로직을 직접 구현했습니다.

### 구현 제약
- 사용자 홈 디렉터리 하위 경로만 처리하도록 제한했습니다.
- `system()` 없이 Linux 시스템 프로그래밍 방식으로 구현했습니다.

### 핵심 구현 상세
- `realpath()`, `stat()`, `scandir()`, `getopt()` 등을 활용해 경로 검증, 디렉터리 순회, 파일 메타데이터 조회, 옵션 파싱 기능을 구현했습니다.
- 상대경로/절대경로를 정규화하고, 홈 디렉터리 외부 접근을 차단하는 경로 검증 로직을 구성했습니다.
- `tree` 명령에서 디렉터리와 파일을 사전순으로 정렬하여 계층 구조 형태로 출력하도록 구현했습니다.
- `arrange` 명령에서 확장자별 디렉터리를 생성하고, 파일을 해당 디렉터리로 분류하도록 구현했습니다.
- 동일한 파일명이 존재하는 경우 사용자 선택에 따라 파일 비교(`diff`), 편집기 확인(`vi`), 특정 파일 선택, 건너뛰기를 제공하는 충돌 처리 로직을 구현했습니다.

### 프로그램 내장 명령어

#### 1. `tree <dir_path> [option]`
지정한 디렉터리의 구조를 사전순으로 정렬하여 재귀적으로 tree 형태로 출력합니다.

| 옵션 | 인자 | 기능 |
| :---: | :---: | :--- |
| `-p` |  | 각 파일 및 디렉터리의 권한 정보를 함께 출력합니다. |
| `-s` |  | 각 파일 및 디렉터리의 크기 정보를 함께 출력합니다. |

#### 2. `arrange <dir_path> [option]`
지정한 디렉터리 내부 파일을 확장자 기준으로 분류하여 정리합니다.

| 옵션 | 인자 | 기능 |
| :---: | :---: | :--- |
| `-d` | `output_path` | 정리 결과를 저장할 디렉터리를 직접 지정합니다. |
| `-t` | `seconds` | 수정 후 일정 시간이 지난 파일만 정리합니다. |
| `-x` | `exclude_path...` | 정리 대상에서 제외할 하위 디렉터리를 지정합니다. |
| `-e` | `extension...` | 특정 확장자를 가진 파일만 선택적으로 정리합니다. |

#### 3. `help`
내장 명령어의 사용법을 출력합니다.

#### 4. `exit`
프로그램을 종료합니다.

## 과제 #2 - ssu_cleanupd
### 개요
사용자가 지정한 디렉터리를 주기적으로 모니터링하고, 새롭게 추가된 파일을 자동으로 정리할 수 있도록 구현한 Linux 기반 CLI 데몬 관리 프로그램입니다.  
데몬 프로세스 생성 및 관리, 경로 검증, 설정 파일 기반 동작 갱신, 로그 기록, 파일 락을 활용한 동시성 제어 로직을 직접 구현했습니다.

### 구현 제약
- 사용자 홈 디렉터리 하위 경로만 처리하도록 제한했습니다.
- `system()` 없이 Linux 시스템 프로그래밍 방식으로 구현했습니다.

### 핵심 구현 상세
- `realpath()`, `scandir()`, `getopt()` 등을 활용해 경로 정규화, 디렉터리 순회, 옵션 파싱 기능을 구현했습니다.
- 프로그램 실행 시 `~/.ssu_cleanupd/current_daemon_list`를 생성·관리하여 현재 실행 중인 데몬 프로세스 목록을 추적하도록 구성했습니다.
- `add` 명령을 통해 지정한 디렉터리를 주기적으로 모니터링하는 데몬 프로세스를 등록하고, 새로 추가된 파일을 확장자별 디렉터리로 자동 분류·복사하도록 구현했습니다.
- 데몬별 설정을 `ssu_cleanupd.config` 파일에 저장하고, 주기마다 해당 파일을 다시 읽어 변경된 옵션이 반영되도록 구성했습니다.
- 파일 정리 이력을 `ssu_cleanupd.log`에 기록하고, 최대 로그 줄 수 제한 옵션을 지원하도록 구현했습니다.
- `modify` 명령 수행 시 설정 파일 접근에 `fcntl()` 기반 파일 락을 적용해, 데몬의 설정 파일 읽기와 사용자 수정 작업이 동시에 발생하지 않도록 처리했습니다.
- 중복 파일명은 모드 값에 따라 최신 파일 선택, 가장 오래된 파일 선택, 정리 제외 중 하나로 처리하도록 구현했습니다.

### 프로그램 내장 명령어

#### 1. `show`
현재 실행 중인 데몬 프로세스 목록을 출력하고, 선택한 데몬의 설정 정보와 최근 로그를 확인합니다.

#### 2. `add <dir_path> [option]`
지정한 디렉터리를 주기적으로 모니터링하는 데몬 프로세스를 등록하고, 정리되지 않은 파일을 자동으로 분류합니다.

| 옵션 | 인자 | 기능 |
| :---: | :---: | :--- |
| `-d` | `output_path` | 정리 결과를 저장할 디렉터리를 직접 지정합니다. |
| `-i` | `time_interval` | 데몬이 디렉터리를 검사하는 주기를 초 단위로 지정합니다. |
| `-l` | `max_log_lines` | 로그 파일의 최대 줄 수를 지정합니다. |
| `-x` | `exclude_path...` | 정리 대상에서 제외할 하위 디렉터리를 지정합니다. |
| `-e` | `extension...` | 특정 확장자를 가진 파일만 선택적으로 정리합니다. |
| `-m` | `mode` | 동일한 파일명이 존재할 때의 처리 방식을 지정합니다. |

#### 3. `modify <dir_path> [option]`
이미 모니터링 중인 디렉터리의 설정 파일을 수정하여, 데몬 프로세스가 새로운 옵션으로 동작하도록 갱신합니다.

| 옵션 | 인자 | 기능 |
| :---: | :---: | :--- |
| `-d` | `output_path` | 정리 결과를 저장할 디렉터리를 직접 지정합니다. |
| `-i` | `time_interval` | 데몬의 검사 주기를 초 단위로 변경합니다. |
| `-l` | `max_log_lines` | 로그 파일의 최대 줄 수를 변경합니다. |
| `-x` | `exclude_path...` | 정리 대상에서 제외할 하위 디렉터리를 지정합니다. |
| `-e` | `extension...` | 특정 확장자를 가진 파일만 선택적으로 정리합니다. |
| `-m` | `mode` | 동일한 파일명이 존재할 때의 처리 방식을 변경합니다. |

#### 4. `remove <dir_path>`
지정한 디렉터리를 모니터링 중인 데몬 프로세스를 종료합니다.

#### 5. `help`
내장 명령어의 사용법을 출력합니다.

#### 6. `exit`
프로그램을 종료합니다.

### 주요 설정 파일
- `~/.ssu_cleanupd/current_daemon_list`  
  현재 실행 중인 데몬 프로세스 목록을 저장합니다.
- `ssu_cleanupd.config`  
  모니터링 경로, PID, 시작 시각, 출력 경로, 주기, 로그 제한, 제외 경로, 확장자, 중복 처리 모드 등의 설정을 저장합니다.
- `ssu_cleanupd.log`  
  파일 정리 시각, 데몬 PID, 원본 파일 절대경로, 정리 후 파일 절대경로를 기록합니다.
